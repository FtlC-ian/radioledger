# Security Policy

RadioLedger handles logbook data, station identity data, external-service credentials, and LoTW workflows. Please report suspected vulnerabilities privately.

## Supported Versions

RadioLedger is pre-1.0. Security fixes are applied to the active development branch and the latest published release artifacts when releases exist.

## Reporting a Vulnerability

Do not open a public issue for a vulnerability.

Use one of these private channels:

- A confidential security issue in the project forge, if available.
- Email `security@radioledger.app` with a concise description, affected component, reproduction steps, and any proof-of-concept details.

Please avoid including real credentials, private keys, production tokens, or unrelated personal data in the report. Redacted logs and minimal test data are preferred.

## What To Expect

The project will acknowledge valid reports, triage severity, and publish fixes or mitigations as quickly as practical. Public disclosure should wait until a fix or mitigation is available unless there is active exploitation or an urgent user-safety reason to disclose sooner.

## Security Documentation

The public security posture is documented at [docs/security.md](docs/security.md). Self-hosting hardening guidance is documented at [docs/self-hosting/security.md](docs/self-hosting/security.md).
