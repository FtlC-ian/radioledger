RadioLedger is an amateur-radio logbook for operators who care about ADIF fidelity, UTC discipline, confirmations, and keeping an exportable copy of their contacts. Run the stack on hardware you control.

## Prerequisites

- A ham radio callsign
- Docker + Docker Compose

### Suggested first QSO fields

Use a real contact if you have one. For a dry run, enter a normal-looking example and delete it afterward.

| Field | Example | Notes |
| --- | --- | --- |
| Callsign | `W1AW` | Type the callsign as worked, including `/P` or other suffixes. |
| UTC date/time | `2026-06-04 19:32Z` | Log in UTC, not local time. |
| Band | `20m` | Frequency can be added too, for example `14.225`. |
| Mode | `SSB` | Digital modes from WSJT-X can be captured later with the desktop client. |
| RST sent / received | `59` / `57` | Use `599` for CW/digital-style reports when appropriate. |

After saving, the contact should appear in your logbook with pending confirmation status until LoTW, QRZ, eQSL, Club Log, or a paper-card workflow confirms it.

## Self-hosted

1. Create a local project folder:

```bash
mkdir ~/radioledger && cd ~/radioledger
```

2. Download the compose assets:

```bash
git clone https://github.com/FtlC-ian/radioledger.git
cd radioledger/docker
./init.sh
```

3. Edit `.env` and set at minimum:

```dotenv
DB_PASSWORD=replace-with-strong-password
# Keep the RADIOLEDGER_MASTER_KEY value generated by ./init.sh.
BASE_URL=http://localhost:3000
```

4. Start the stack:

```bash
docker compose up -d
```

5. Verify API health:

```bash
curl http://localhost:8080/health
curl http://localhost:8080/ready
```

6. Open `http://localhost:3000` and create your account.

## Troubleshooting

- **Containers restart repeatedly:** check `docker compose logs api` and `docker compose logs db`.
- **`/ready` returns non-OK:** migrations may still be running; wait ~30 seconds and retry.
- **Can't sign in:** confirm `BASE_URL` and auth redirect settings match your deployment URL.
